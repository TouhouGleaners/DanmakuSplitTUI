# DanmakuSplitTUI

基于时间轴排序的 B 站弹幕 XML 分割工具 (TUI 版)。

## 安装

```bash
# 创建虚拟环境 (可选)
python -m venv .venv
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/macOS

# 安装依赖
pip install textual
```

## 使用

```bash
python main.py
```

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `P` | 预览分割结果 |
| `Ctrl+S` | 执行切割 |
| `Q` | 退出程序 |
| `Tab` | 切换输入框 |

### 操作流程

1. **输入文件路径**: 在第一个输入框输入 XML 文件的完整路径
2. **填写负责人**: 用逗号或空格分隔多个负责人姓名 (可选)
3. **选择分割方式**:
   - **按人头平分**: 根据负责人数量平均分配弹幕
   - **按数量限制**: 指定单片弹幕上限
4. **设置参数**:
   - 单片上限: 按数量分割时的每片弹幕数 (留空自动计算)
   - 分P号: 对应视频的分P编号 (默认为 1)
5. **预览**: 按 `P` 或点击预览按钮查看分配结果
6. **执行**: 按 `Ctrl+S` 或点击切割按钮开始处理

### 文件命名规则

- **分配给人时**: `P1_Part1_[负责人].xml`
- **纯按数切时**: `P1_Part1.xml`

### 输出目录

程序会在 XML 文件同级目录创建 `Split_文件名` 文件夹存放切割结果。

## 标准协作流程 (SOP)

1. **切割**: 启动程序，填写参数，执行切割
2. **存证**: 截图终端显示的**任务分配表**并留存
3. **分发**: 根据文件名后缀 `_[负责人].xml` 进行分发
4. **校验**: 统筹人核对表中**时间区间**的**起始锚点**弹幕内容是否一致

## 示例

### 按人头平分

1. 输入文件: `P1.xml`
2. 负责人: `张三, 李四, 王五`
3. 分割方式: 按人头平分
4. 分P号: `1`

结果:
- `P1_Part1_[张三].xml`
- `P1_Part2_[李四].xml`
- `P1_Part3_[王五].xml`

### 按数量限制

1. 输入文件: `P1.xml`
2. 分割方式: 按数量限制
3. 单片上限: `3000`
4. 分P号: `1`

结果:
- `P1_Part1.xml`
- `P1_Part2.xml`
- ...

## 故障排除

### 找不到文件

确保输入的路径正确，可以使用绝对路径或相对路径。

### 分割失败

检查 XML 文件格式是否正确，程序只支持标准的 B 站弹幕 XML 格式。

### TUI 显示异常

确保终端支持 ANSI 颜色和 Unicode 字符。推荐使用 Windows Terminal、iTerm2 或现代终端模拟器。
